# Specs

Engineering specs for major system components. One subdirectory per component, `main.md` as entry point.

Full table of contents with statuses: `README.md` (this directory).

**When adding or changing specs, update `README.md` — it's the single source of truth for the specs index.**

Specs are authoritative for implementation details. Code must match the spec, or the spec must be updated to match the code. No silent drift.

Before implementing a component, write or update its spec first. Spec is code.
