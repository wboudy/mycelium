## Key Takeaways

This paper offers an elegant alternative to the experience replay approach used by [[deep Q-networks learn control policies directly from raw pixels by combining CNNs with experience replay]]. The core problem is the same — online RL generates correlated, non-stationary data that destabilizes neural network training. DQN solved this with a replay buffer (off-policy). A3C solves it with parallel actors (on-policy): multiple agents running simultaneously in separate environment copies naturally decorrelate the data, because at any given moment they're in different states exploring different parts of the state space.

This is a profound insight because it unlocks on-policy methods for deep RL. Experience replay forces you to use off-policy algorithms (you're learning from data generated by an older policy). Asynchronous parallelism removes that constraint — Sarsa, n-step methods, and actor-critic methods all become viable with deep networks. The best variant, Asynchronous Advantage Actor-Critic (A3C), maintains both a policy (actor) and value function (critic) and uses the advantage function (reward minus value baseline) to reduce variance in policy gradient updates.

The practical result is remarkable: A3C on a single multi-core CPU outperforms DQN on a GPU on many Atari games while training in half the time. No specialized hardware, no massive distributed systems (unlike Google's Gorila which used 130 machines). The approach also generalizes to continuous control and 3D maze navigation from visual input — something DQN couldn't handle since it requires discrete action spaces.

The n-step returns are key to making actor-critic work well: instead of bootstrapping from a single step, the agent accumulates rewards over n steps before bootstrapping, propagating reward information faster through the state-action space.

## Claims

- **[causal]** Running multiple parallel agents in separate environment instances decorrelates training data and stabilizes deep RL without requiring experience replay, because agents simultaneously occupy diverse states (supports)
- **[empirical]** A3C on a single multi-core CPU surpasses DQN on GPU on many Atari games while training in half the time and using far fewer resources than distributed approaches (supports)
- **[causal]** Removing the experience replay requirement enables on-policy methods (Sarsa, n-step, actor-critic) to work with deep networks, whereas replay forces off-policy learning (supports)
- **[empirical]** A3C generalizes across 2D Atari games, continuous motor control, and 3D maze navigation from visual input — the most general RL agent at time of publication (supports)
- **[definition]** A3C (Asynchronous Advantage Actor-Critic) uses parallel actor-learners that each compute gradients locally and asynchronously update shared parameters, with advantage estimates (R - V) reducing policy gradient variance (neutral)
- **[causal]** N-step returns propagate reward information to n preceding state-action pairs in a single update rather than requiring iterative one-step bootstrapping, significantly accelerating learning (supports)
- **[empirical]** The asynchronous framework stabilizes all four tested RL algorithms (one-step Q-learning, n-step Q-learning, Sarsa, actor-critic), not just actor-critic (supports)

> Abstract: We propose a conceptually simple and lightweight framework for deep reinforcement learning that uses asynchronous gradient descent for optimization of deep neural network controllers. We present asynchronous variants of four standard reinforcement learning algorithms and show that parallel actor-learners have a stabilizing effect on training allowing all four methods to successfully train neural network controllers. The best performing method, an asynchronous variant of actor-critic, surpasses the current state-of-the-art on the Atari domain while training for half the time on a single multi-core CPU instead of a GPU.
